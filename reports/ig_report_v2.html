TL;DR — what the eval shows

The pipeline runs end-to-end on Instagram with zero code changes beyond the source step — download → twin → voice → looks → render → report all succeeded on every account. The variable that decides quality is the source footage, not the platform.

Two gotchas that matter for social video. (1) Pick the right A-roll. A creator's top Reels are often skits — search deeper for a forward-facing talking-head, exactly like the YouTube harvest (Ryan was unusable until we did). (2) Captions bleed in. Avatar V is cross-reference-driven, so it reproduces the source scene including burned-in captions; short-form is almost always captioned, so the caption-free Image-N looks are the recommended deliverable.

What we learned

  • Platform is not the variable — source footage is. The IG pipeline is byte-for-byte the YouTube one past the download step; every account trained + cloned + rendered successfully.
  • Burned-in captions are the #1 social-footage artifact. Avatar V animates the actual footage, so caption text rides into the real-look twin. Generated Image-N looks are caption-free and side-step it entirely.
  • Identity fidelity scales with the reference. A clean, forward, well-lit face → ArcFace ~0.85. A down-angle phone-skit → ~0.51. Same code, very different output.
  • Finding the window is the human-in-the-loop step. Big creators (Ryan, 3.2M) can be harder than niche ones (David, 9K) because their content is over-produced. Worth pre-screening profiles for at least one clean talking-head Reel.

9Method

Per account: yt-dlp pulls the Reel → pick a clean continuous window from a contact sheet → extract a native-9:16 twin clip + voice sample → upload + create an Avatar V digital twin via /v2/video_avatar with a pre-cleared R&D consent video bundled → clone the voice → generate two Image-N looks (Seedance, identity-scored with ArcFace) → render a 4-beat emotion arc (ElevenLabs v3, per-beat motion prompts) on both the real look and a generated look → host + report. Renders are internal HeyGen R&D on the operator's own account.

Skills: av-instagram-twin / av-tiktok-twin (adapted from av-emotional-twin). Identity = ArcFace cosine vs a source reference frame; never a VLM judge.
